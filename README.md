
# Лабораторная работа 3

Проект демонстрирует собственный динамический пул потоков для запуска тестов без `ThreadPool` и `Task.Run`.

## Структура

1. `MiniTestFramework/` — библиотека тестирования и раннер.
2. `TestedProject/` — тестируемые классы.
3. `Tests/` — набор тестов.
4. `DynamicThreadPool/` — собственный пул потоков на `Thread` + `Monitor`.
5. `Program.cs` — программа моделирования неравномерной нагрузки и сравнения режимов.

## Что реализовано

- настраиваемые границы `MinWorkerCount` / `MaxWorkerCount`;
- автоматическое масштабирование при росте очереди и времени ожидания;
- адаптивное сжатие простаивающих потоков;
- мониторинг состояния пула и очереди в live-логе;
- логирование ошибок внутри воркеров;
- восстановление мощности пула при сбоях воркеров;
- механизм замены подозрительно зависших воркеров;
- сценарий нагрузки с паузами, одиночными подачами и пиковыми всплесками;
- не менее `60` запусков тестов за один сценарий.

## Запуск

```bash
dotnet build
dotnet run
dotnet run -- 2 8
```

Аргументы:

- `args[0]` — минимальное число потоков и размер фиксированного пула для базового сравнения.
- `args[1]` — максимальное число потоков для динамического режима.
